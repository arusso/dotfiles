# Neovim Configuration

This directory contains the Neovim configuration using Lua and lazy.nvim for plugin management.

## Directory Structure

```
nvim/
├── init.lua              # Entry point - loads config in order
├── lazy-lock.json        # Locked plugin versions (generated by lazy.nvim)
└── lua/
    ├── config/           # Core configuration
    │   ├── init.lua      # Bootstraps lazy.nvim and loads plugins
    │   ├── options.lua   # Global options (leader keys, clipboard, etc.)
    │   ├── keymaps.lua   # Keybindings
    │   └── lazy.lua      # Lazy.nvim bootstrap and setup
    └── plugins/          # Plugin specifications
        ├── git.lua       # Git-related plugins (git-blame)
        ├── formatting.lua # Format-on-save configuration
        ├── languages.lua  # Language-specific plugins
        ├── lsp.lua        # LSP configuration (lua_ls)
        ├── ui.lua         # UI plugins (nvim-tree, Comment.nvim)
        └── solarized.lua  # Colorscheme configuration
```

## Architecture

### Loading Order

1. `init.lua` loads config in this order:
   - `config/options.lua` - Sets global options and leader keys
   - `config/keymaps.lua` - Sets up keybindings
   - `config/lazy.lua` - Bootstraps lazy.nvim and loads plugins

2. lazy.nvim loads plugins from the `plugins/` directory

### Key Configuration

- **Leader key**: `-`
- **Local leader**: `\`
- **Tab width**: 2 spaces
- **Clipboard**: `unnamedplus` (system clipboard)
- **Colorscheme**: solarized (dark)

## Adding New Plugins

### Step 1: Add to Plugin Spec

Create or edit a file in `lua/plugins/`:

```lua
return {
  { "author/plugin-name.lua" },
}
```

For plugins requiring configuration:

```lua
return {
  {
    "author/plugin-name.lua",
    config = function()
      require("plugin-name").setup({
        -- options here
      })
    end
  }
}
```

Or with options table:

```lua
return {
  {
    "author/plugin-name.lua",
    opts = {
      -- options here
    },
  }
}
```

### Step 2: Lock Plugin Version

After adding the plugin, open Neovim and run:

```
:Mason
```

Or run in terminal:
```bash
nvim --headless +LazySync +qa
```

This updates `lazy-lock.json` with the specific commit hash.

### Plugin Organization Guidelines

- **UI plugins** (file explorer, statusline, etc.) → `ui.lua`
- **Language support** (vim-go, ansible-vim, etc.) → `languages.lua`
- **LSP configurations** → `lsp.lua`
- **Formatting tools** → `formatting.lua`
- **Git tools** → `git.lua`

## Common Tasks

### Change Leader Key

Edit `lua/config/options.lua`:
```lua
vim.g.mapleader = ','  -- change to comma
```

### Add Keybinding

Edit `lua/config/keymaps.lua`:
```lua
vim.keymap.set('n', '<leader>w', ':w<cr>', { desc = 'Save file' })
```

### Disable a Plugin

Comment out or remove the plugin from its spec file, then run `:Lazy sync`.
